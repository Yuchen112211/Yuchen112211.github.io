Currently the game looks pretty easy(in design, not game itself), if given more time and resource, I think I can optimize the game in following aspects.

Adding a user id system.The game now keeps everything inside the memory, it does not interact with database system. Adding a database in the back of the game can ensure players keeping their scores, maintaining user identity and optimize the experience of the players. Thought this requires a lot of work, especially we have to integrate C++ with MySQL or MongoDB for backend support, but if we want the game to expand or attract more people this function definitely matters.

Adding more language.Currently the game only contains two languages, English and Italian, which is not enough. More language can attract more users.

Design more patterns.Like special effect, after 10 bumps the ball turns into a bigger ball, or turn into a bomb that blows up an area of bricks. Different effects can make the game more playable, avoid players getting bored.
